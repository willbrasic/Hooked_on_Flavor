# 4th Year Paper Project

## R Preferences
- Use `data.table` syntax
- Comment code explaining what each section does
- Use `pacman::p_load()` for loading packages
- Avoid scientific notation: `options(scipen = 999)`
- Put `{` on new line for `for`/`if` blocks
- Use `keyby` instead of `by` + `setorder` when ordering is needed

## Code Style
- Clear section headers with `############################`
- Descriptive variable names
- Inline comments explaining non-obvious logic
- Start lists and data tables using "list_" and "dt_", where a description follows the underscore
